* Lab 4

Lab 4, part 1 was a favorite of mine because we started working with multiple files and linking them together. I found this amazing as it opens up a ton of possibilities especially when making large complicated programs. It gives you the option to make small modular bits of code and combine them all into a larger system to make one cohesive program. It allows multiple people from different areas to tackle small specific problems to improve a system as a whole. Even if you’re working on a program alone it allows you to break up a complicated problem and solve it bit by bit and reduce a lengthy debugging process.


* Lab 11

Lab 11, part 1 was a favorite of mine because we worked with defining classes and learned about object-oriented programming. I thought this was quite interesting as in some cases you have to rationalize or define an abstract idea. Given this lab as an example how do you define and rationalize what a U.S. state is? Learning about object-oriented programming also made me think of a different perspective when it comes to interacting with the world and the program itself. It shifted my thinking from everything being an inert object to something that can interact back with me for example a red pen knows it’s red and communicates that back to me.
